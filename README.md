## Overview

This repository contains the full replication code for Study 2 of a two-paper series on evaluator upgrade overrides in corporate credit rating.

**Study 1** (companion paper) established that upgrade overrides are systematically associated with elevated default rates using the Polish Companies Bankruptcy dataset.

**Study 2 (this repository)** asks the deeper question: *which specific financial patterns do evaluators overlook when they upgrade a firm's grade, and can these be formalised into a pre-emptive warning system?*

The answer is a **two-layer warning factor library** — 50 single-variable factors (Layer 1) and 115 composite-pattern rules (Layer 2) — that flags high-risk upgrade decisions in real time, before the outcome is known.

```
data/raw/1year.arff
```

## Notebook Descriptions

### NB01 — Data Preparation
Loads the five-year `.arff` files, trains the system logistic regression model (C=0.1, 5-fold CV, ROC-AUC = 0.778), assigns quantile-based grades (AAA–CCC), simulates the Base scenario override distribution, and constructs the `upgrade_cohort` (N = 15,684) with `risky_upgrade` / `safe_upgrade` group labels.

**Outputs:** `data/processed/polish_master.parquet`, `data/processed/upgrade_cohort.parquet`

---

### NB02 — Exploratory Data Analysis
Applies Mann-Whitney U tests (two-sided) to all 64 financial variables comparing `risky_upgrade` vs `safe_upgrade`. Produces volcano plot, box plots (top 20), distribution histograms (top 6), and correlation heatmap for significant variables.

**Key result:** 58 / 64 variables significant at p < 0.05; 54 at p < 0.001. All differences negative (risky firms weaker on all dimensions).

**Outputs:** `results/tables/NB02_01_descriptive_stats.csv`, `NB02_02_mannwhitney_results.csv`

---

### NB03 — Univariate Firth Logistic Regression
Applies Firth penalised logistic regression to all 58 Mann-Whitney-significant variables. FDR correction (Benjamini-Hochberg) applied across all 58 tests. VIF screening (threshold = 10) removes 34 collinear variables. Effect sizes quantified via rank-biserial correlation and Cohen's d.

**Key result:** 50 variables pass FDR q < 0.05 → Layer 1 factor pool. 16 variables pass VIF screening → ARM candidate set.

**Outputs:** `results/tables/NB03_01_univariate_logistic.csv`, `NB03_02_top_candidates.csv`, `NB03_03_vif_screening.csv`

---

### NB03b — Data Dictionary Check
Maps Attr-coded variable names to their UCI definitions. Confirms no near-duplicate pairs (|r| > 0.98) among the 16 ARM candidates.

**Outputs:** `results/tables/NB03b_variable_map.csv`

---

### NB04 — Association Rule Mining
Discretises 16 candidate variables into tertile bins (LOW/MID/HIGH). Applies Apriori to `risky_upgrade` and `safe_upgrade` cohorts separately. Filters risky-specific rules: support ≥ 0.20, confidence ≥ 0.60, lift ≥ 2.0×, conf_diff ≥ 0.10, antecedent max length = 3.

**Key result:** 491 risky-specific rules retained. Top rule (Attr55=LOW & Attr58=HIGH): confidence = 0.942, lift = 11.29×.

**Outputs:** `results/tables/NB04_01_rules_risky.csv`, `NB04_02_rules_summary.csv`, `NB04_03_top10_rules.csv`

---

### NB05 — K-Means Clustering
Clusters 1,308 `risky_upgrade` firms using K-Means. Optimal K selected via combined Silhouette + Davies-Bouldin score with minimum cluster size constraint (≥ 5% of sample). Fallback to Elbow (K = 4) when constraint cannot be satisfied.

**Key result:** K = 4 clusters. Dominant cluster (n = 743, 56.8%): chronic profitability deficit. Cluster 0 (n = 191): extreme collapse. Cluster 3 (n = 349): asset-turnover and margin deterioration.

**Outputs:** `results/tables/NB05_01_cluster_profiles.csv`, `NB05_02_cluster_assignments.csv`, `NB05_03_k_selection.csv`

---

### NB06 — Warning Factor Library
Constructs the two-layer factor library:
- **Layer 1** (50 factors): all FDR-significant variables, trigger = distress tertile
- **Layer 2** (115 factors): risky-specific rules with confidence ≥ 0.75 and lift ≥ 2.0×, deduplicated by unique antecedent

Also produces the system design specification CSV documenting the full deployment architecture.

**Outputs:** `NB06_L1_single_factors.csv`, `NB06_L2_composite_factors.csv`, `NB06_full_factor_library.csv`, `NB06_03_system_design_spec.csv`

---

### NB07 — Retrospective Simulation
Applies all 165 warning factors to all 15,684 firms and answers five operational questions:

| Q | Question | Key finding |
|---|----------|-------------|
| Q1 | Warning count vs default rate | 50+ warnings → 12.97% DR (4.9× no-warning baseline) |
| Q2 | Which factors were most accurate? | L2-91 (Attr12+Attr24+Attr55=LOW): hit rate 16.6%, lift 1.99× |
| Q3 | Coverage-precision tradeoff | ≥1 warning: 98.7% coverage; ≥20: 11.6% precision |
| Q4 | Where does system work best? | CCC→B: F1=0.281, recall=93.1%; Year 5: F1=0.388 |
| Q5 | Cost of ignoring warnings | 30+ warnings → 4.64× higher DR than no-warning |

**Outputs:** `NB07_01_firm_level_warnings.csv`, `NB07_Q1` through `NB07_Q5` tables

## Reproducibility

All random states are fixed (`RANDOM_SEED = 42`). Results are fully reproducible given the same input data and package versions. The override simulation uses `np.random.default_rng(42)`.

Minor numerical differences may occur across operating systems due to floating-point handling in `scipy` and `sklearn`.
